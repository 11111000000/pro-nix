# Инструкция для developer-агента

## Назначение
- Агент вносит минимальные, проверяемые изменения в pro-nix.
- Агент работает в рамках политики `AGENTS.md` и не дублирует её.

## Перед изменением
1. Прочитать `AGENTS.md`, `SURFACE.md`, `HOLO.md`, `README.md`.
2. Определить один Intent.
3. Проверить, затрагивается ли публичное поведение.
4. Если да, подготовить Change Gate.

## Формат ответа
- Краткое резюме.
- Change Gate.
- Список файлов.
- Патч.
- Проверка.

## Правила качества
- Не расширять объём изменения без нужды.
- Не менять нерелевантные файлы.
- Не смешивать рефакторинг с новым поведением.
- Для Nix: модульный вклад через `lib.mkDefault`, окончательная фиксация только на уровне host-конфигурации.
- Для Emacs Lisp: одна функция — одна задача, публичные функции с docstring, поведение с ERT.

## Проверки
- `./tools/surface-lint.sh`
- `./tools/holo-verify.sh`
- `nix fmt`
- `nix flake check`
- headless ERT для Emacs-изменений

## Отказ
- Если изменение затрагивает `SURFACE.md` с `[FROZEN]`, а Change Gate или Proof отсутствуют, агент должен остановиться.
