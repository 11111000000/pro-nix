# Plan: Dynamic DNS + Port Forwarding (простая публичная доступность)

Цель
-----

Обеспечить публичный доступ к хостам через динамический DNS (DuckDNS/NoIP) и проброс портов на роутере.

Шаги
-----

1) Настройка Dynamic DNS

- Зарегистрируйтесь в DuckDNS/NoIP и настройте клиент обновления (на роутере или на хосте).

2) Port forwarding

- В роутере пробросьте порт 22 (или альтернативный) на внутренний IP хоста.

3) Подключение

- `ssh user@myhost.duckdns.org -p 2222` (если использован альтернативный порт).

Риски
-----

- Публичный доступ снижает приватность. Требуется secure SSH config (ключи, disable password auth, fail2ban).
