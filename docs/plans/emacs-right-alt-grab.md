# Emacs: локальный перехват Right-Alt (AltGr) при фокусе

Цель
-----
Обеспечить поведение: когда окно Emacs в фокусе, нажатие Right‑Alt (AltGr)
выполняет `toggle-input-method` в Emacs, при этом во всех остальных окнах
Right‑Alt остаётся системным переключателем раскладки. Решение должно быть
безопасным, минимально инвазивным и обратимым.

Короткая архитектура
--------------------
Подход основывается на внешнем пользовательском демоне (helper), который
следит за активным окном в X11 сессии и временно грабит/освобождает
клавишу Right‑Alt: когда Emacs — grab установлен и при нажатии отправляется
команда в Emacs (через `emacsclient --eval "(toggle-input-method)"`);
когда фокус уходит — grab снимается и поведение возвращается системе.

Почему этот подход
------------------
- Не меняет глобальные XKB/DE настройки навсегда — поведение локально и
  обратимо.
- Работает без постоянных прав администратора.
- Не трогает пользователя в других приложениях.

Ограничения и варианты
-----------------------
- X11 vs Wayland:
  - X11: предлагаемая схема надёжна (python-xlib или аналог); можно перехватывать
    клавиши через XGrabKey и слушать _NET_ACTIVE_WINDOW.
  - Wayland: глобальные key‑grabs запрещены в большинстве compositors; для
    sway возможен workaround через swaymsg (подписка на события и изменение
    xkb_options), для GNOME/Mutter аналогичного универсального способа нет.
- Терминал: терминальный эмулятор обычно обрабатывает AltGr; Emacs в tty
  не получит сырой Right‑Alt; для tty предложить альтернативную клавишу
  (например, C-`).

Детализованный план (X11)
-------------------------
1. Требования
   - Emacs GUI (не tty) под X11 или XWayland.
   - Наличие `emacsclient` (Emacs server) либо готовность запустить Emacs
     как daemon; helper будет вызывать emacsclient для переключения.
   - Python 3 + пакет python-xlib (или системный эквивалент).

2. Компоненты
   - helper daemon (python):
     - слушает изменение активного окна (_NET_ACTIVE_WINDOW) на root window;
     - на FocusIn проверяет WM_CLASS/WM_NAME окна — если это Emacs, вызывает
       XGrabKey для keysym Right‑Alt/ISO_Level3_Shift;
     - на FocusOut/XLib event снимает grab (XUngrabKey);
     - на KeyPress для граббера запускает `emacsclient --eval "(toggle-input-method)"`;
     - аккуратно обрабатывает модификаторы (опция: реагировать только когда
       нажата только Right‑Alt без Ctrl/Shift/Super);
     - логирует ошибки и корректно очищает grab при выходе.

   - systemd --user unit (опционально): автозапуск демона при входе в сессию.

   - Emacs config snippet (init.el):
     - резервы: backup key (например, C-`) для tty и случаев, когда helper не
       доступен;
     - инструкции для проверки имени события (C-h k Right‑Alt) — отладка.

3. Поведение при нажатии
   - helper запускает `emacsclient --eval '(toggle-input-method)'`.
   - helper подавляет дальнейшую обработку события в X (не ретранслирует
     на систему), поэтому в Emacs переключение выполняется и системная
     раскладка не меняется.

4. Защита от edge-cases
   - Если Emacs не отвечает/нет emacsclient — helper выводит сообщение в лог
     и ничего не ломает (grab остаётся, но KeyPress не приводит к crash);
     при желании можно сделать fallback: воспроизвести локальный индикатор и
     отпустить grab.
   - Если пользователь привык использовать AltGr для ввода символов в
     Emacs (AltGr+символ) — это поведение будет блокировано. Опция: в
     helper добавить режим, когда при удержании AltGr+другой клавиши
     событие передаётся дальше (сложно в реализации); проще документировать
     побочный эффект.

5. Тестирование
   - ручные проверки:
     - запустить helper, активировать Emacs, нажать Right‑Alt — должна
       выполниться toggle-input-method;
     - переключиться в другое окно — Right‑Alt должен снова переключать
       системную раскладку;
     - проверить сочетания с Shift/Ctrl — поведение должно быть ожидаемым;
     - убедиться, что при аварийном завершении helper grab снимается.

Wayland (sway) — альтернативная схема
--------------------------------------
- Для sway можно подписываться на события окон (swaymsg -t subscribe '"["window"]"')
  и при фокусе Emacs временно менять xkb_options для устройства ввода
  (swaymsg input <id> xkb_options "<новые опции>"). Это переключение не
  даёт эквивалентного права на перехват клавиши, но позволяет при фокусе
  убрать системный RAlt→grp опцию; на уходе — восстановить её.
- Минусы: требует прав доступа к swaymsg и аккуратного хранения/восстановления
  прежних опций; возможна видимая миграция раскладки при смене фокуса.

Терминал (tty) — поведение
-------------------------
- В tty/терминале перехват Right‑Alt реализовать нельзя надёжно; рекомендуем
  добавить резервную клавишу (C-`) в Emacs конфиге, чтобы переключение
  работало и в tty.

Примеры/шаблоны (для реализации)
---------------------------------
- Emacs init.el snippet:

  ;; fallback для tty
  (global-set-key (kbd "C-`") #'toggle-input-method)

  ;; под GUI дополнительно можно назначить прямо (после проверки имени):
  ;; (global-set-key (kbd "<right-alt>") #'toggle-input-method)

- systemd --user unit (пример в репозитории при реализации):

  [Unit]
  Description=Emacs RightAlt helper

  [Service]
  ExecStart=/usr/bin/python3 /home/user/.local/bin/emacs-rightalt-helper.py
  Restart=on-failure

  [Install]
  WantedBy=default.target

Преимущества/Риски
------------------
- Плюсы: минимальный побочный эффект, поведение только при фокусе Emacs,
  не ломает системные настройки навсегда.
- Минусы: для Wayland и tty решение ограничено; блокировка AltGr убирает
  возможность вводить AltGr-символы в Emacs (если это важно — обсуждаем
  более сложное поведение). Нужен Emacs server (emacsclient) для удобной
  интеграции.
