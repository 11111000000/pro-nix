# Install Matrix

## Purpose

Explain the exact commands for each supported environment without forcing one installation path on everyone.

## 1. New NixOS machine

Use when the machine is not yet configured.

### Commands

```bash
sudo nixos-generate-config
sudo nixos-rebuild switch --flake .#default
```

### Notes
- `hardware-configuration.nix` is generated by NixOS and should be reviewed, not blindly copied.
- The bootstrap script is optional here; manual `nixos-rebuild` is the most direct path.

## 2. Predefined NixOS host

Use when the machine already matches one of the repo profiles.

### Commands

```bash
sudo nixos-rebuild switch --flake .#thinkpad
sudo nixos-rebuild switch --flake .#desktop
sudo nixos-rebuild switch --flake .#cf19
```

### Notes
- `default` maps to the current primary NixOS host.
- `./bootstrap/install.sh` can be used as an interactive wrapper.

## 3. Portable Emacs with Home Manager

Use when you want the Emacs layer without the whole NixOS system.

### Commands

```nix
imports = [ /path/to/repo/emacs/home-manager.nix ];
pro.emacs.enable = true;
pro.emacs.gui.enable = false;
```

Then apply with your Home Manager workflow.

### Notes
- This path works for Linux, WSL, and Termux if Home Manager is available.
- Set `gui.enable = true` only when the environment actually provides a GUI stack.

## 4. Plain `~/.emacs.d`

Use when you want no Nix integration at all.

### Commands

```bash
./scripts/emacs-sync.sh ~/.emacs.d
```

### Notes
- This copies the portable loader and module tree into the target directory.
- It is the simplest path for WSL without Nix, Termux without Nix, or any plain Linux host.

## 5. WSL

### With Nix
- Use the Home Manager profile.
- Prefer `gui.enable = false` unless WSLg or another GUI path is present.

### Without Nix
- Use `./scripts/emacs-sync.sh ~/.emacs.d`.
- Launch Emacs directly from the terminal.

## 6. Termux

### With Nix
- Use the Home Manager profile if your setup supports it.
- Keep `gui.enable = false` by default.

### Without Nix
- Use `./scripts/emacs-sync.sh ~/.emacs.d`.
- Do not assume desktop integration or X resources.

## Verification

After any install path:

```bash
just flake-check
just headless
```

For full machine coverage:

```bash
just check-all
```
