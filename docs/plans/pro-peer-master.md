# Pro‑peer master — мастер безопасной настройки и распространения ключей

Цель
-----

Дать простую, надёжную и объяснённую по‑русски процедуру (и вспомогательный скрипт) для оператора, чтобы подготовить и безопасно распространить зашифрованный файл `authorized_keys.gpg` на хосты pro‑nix, проверить базовую сетевую доступность и аккуратно запустить синхронизацию ключей.

Принципы
---------

- Никаких изменений в основной Nix конфигурации: мастер не меняет nix files и не выполняет `nixos-rebuild`.
- Идемпотентность: повторный запуск мастер‑скрипта не ломает систему; перед перезаписью делается резервная копия.
- Безопасность: приватные ключи не хранятся в репозитории; мастер работает с уже зашифрованным файлом или шифрует локально по явной команде.
- Ясность: каждый шаг сопровождается пояснением на русском и возможностью отмены.

Где находится
------------

- Скрипт: `scripts/pro-peer-master.sh`
- Документация: этот файл.

Короткая инструкция пользователя (обзор шагов)
-------------------------------------------

1. Подготовьте plaintext `authorized_keys` (список публичных ключей доверенных операторов).
2. Защитите его GPG‑шифрованием: `gpg --encrypt --recipient '<GPG_ID>' -o authorized_keys.gpg authorized_keys`.
3. Запустите мастер: `scripts/pro-peer-master.sh --hosts cf19.local,huawei.local --file ./authorized_keys.gpg`.
4. Мастер проверит связь с каждым хостом, загрузит `.gpg` во временное место (в `/tmp`), сделает резервную копию старого файла (если есть), атомарно заменит файл в `/etc/pro-peer/authorized_keys.gpg` и перезапустит сервис синхронизации через `systemctl`.
5. Мастер выведет краткий отчёт по каждому хосту и журнал вызова systemd.

Дополнительные флаги мастера:

- `--dry-run` — показать действия без выполнения;
- `--yes` — ответить "да" на все подтверждения и работать non-interactive;
- `--retries N` — число попыток SSH/SCP (по умолчанию 3);
- `--timeout S` — timeout для SSH/SCP в секундах (по умолчанию 10);
- `--auto-rollback` — при ошибке синхронизации автоматически пытается откатить предыдущую версию;
- `--sig /path/to/file.sig` — проверить detached signature перед распространением;
- `--verify-recipient <GPG_ID>` — проверить, что зашифрованный файл адресован определённому GPG recipient.

Fail‑safe поведение
-------------------

- Если загрузка/перемещение файла не удалась — мастер откатывает изменения и оставляет старую версию в целевом месте.
- Мастер не задействует `nixos-rebuild` и не меняет сервисы кроме вызова `systemctl restart pro-peer-sync-keys.service`.
- Все операции логируются в local stdout и (по желанию) в файл `~/.local/share/pro-peer/master.log`.

Предусловия
------------

- На операторской машине: `ssh`, `scp`, `gpg` установлены.
- На целевых хостах: `ssh` доступен, sudo разрешён для администратора, на хостах настроен `pro-peer` модуль (sync service exists).

Далее — подробный пошаговый мастер и примеры команд см. скрипт `scripts/pro-peer-master.sh`.
