# Keys Workflow (Global Keys via Org)

Политика
- Все глобальные клавиши задаются и документируются в Org-таблицах:
  - Системный слой: `emacs-keys.org` в корне репозитория
  - Пользовательский слой: `~/.config/emacs/keys.org`
- Модули не назначают глобальные бинды напрямую (никаких `global-set-key` вне `keys.el`).
- Локальные бинды в mode-map допустимы (например, `(define-key org-mode-map ...)`).

Как добавить/изменить глобальную клавишу
1. Откройте ключевой файл (system или user):
   - System: `repo/emacs-keys.org`
   - User: `M-x pro-org-open-keys-file` (откроет `~/.config/emacs/keys.org`)
2. Добавьте строку в таблицу формата: `| Секция | Клавиша | Команда | Примечание |`
3. Примените изменения: `M-x pro-keys-reload`

Слои и приоритеты
- Сначала загружается системный файл (`emacs-keys.org`), затем пользовательский (`~/.config/emacs/keys.org`).
- Пользовательский слой перекрывает системный при совпадении комбинации.

EXWM и Org-mode
- Строки в секции `EXWM` накапливаются в `exwm-input-global-keys`.
- Строки в секции `ORG` применяются к `org-mode-map` (после загрузки org-mode).

Диагностика
- `M-x pro-keys-reload` — перезагрузить ключи из обоих файлов и применить отложенные бинды, если команды уже загружены.
- `M-x pro-keys-report-pending` — показать незадействованные бинды (команды ещё не загружены).

Откат
- Поскольку файлы — обычный текст, используйте историю git (для системного слоя) и резервные копии/редактирование (для пользовательского слоя).

Линт
- CI-скрипт `scripts/lint-keys.sh` гарантирует, что `global-set-key` не используется в модулях напрямую. Это фиксирует правило: «глобальные клавиши только через Org».
