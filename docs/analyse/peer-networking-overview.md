# Обзор всех путей для peer‑discovery и доступа

Этот документ агрегирует и сравнивает все рассмотренные варианты обеспечения доступности машин pro‑nix друг для друга по именам как в локальной сети, так и через Интернет.

Структура

- Краткая сводка вариантов
- Как выбрать подходящий путь
- Сравнительная таблица рисков и ограничений

Варианты (кратко)

1. Avahi / mDNS (локальная сеть)
   - Нулевой конфиг для LAN. Разрешает host.local имена в одной L2 сети.
   - Не работает через роутеры/интернет.

2. Tor Hidden Services
   - Работает через NAT и файрволлы без порт‑форвардинга.
   - Даёт .onion адреса для SSH/HTTP и т.д.; скрывает реальный IP.
   - Высокая латентность, но прост в разворачивании.

3. I2P
   - P2P ориентированное перекрытие, менее распространено чем Tor.
   - Более сложная настройка и меньшая экосистема.

4. Yggdrasil (IPv6 mesh)
   - Самоорганизующаяся IPv6 overlay сеть; узлы получают стабильные IPv6 адреса.
   - Подходит для автономного mesh без центра.

5. WireGuard + Headscale / Nebula
   - WireGuard даёт производительный VPN; Headscale — self‑hosted control plane (аналог Tailscale без облака).
   - Нужен как минимум один control plane (можно держать на VPS или NAS).

6. Dynamic DNS + Port Forwarding
   - Простой публичный доступ через DuckDNS/NoIP + проброс портов на роутере.
   - Требует публичного IP/ручного проброса портов; менее приватно.

7. Proxy / Local binary cache (ncps / nginx)
   - Не имя‑разрешение само по себе, но помогает распространению данных (cache) в LAN.
   - Вариант для распространения артефактов, не для SSH‑reachability.

Как выбирать

- Если цель — простая локальная доступность (машины в одной Wi‑Fi): Avahi.
- Если цель — доступ извне через интернет без проброса портов и с приватностью: Tor Hidden Services.
- Если цель — производительный и стабильный mesh между часто‑включёнными машинами: WireGuard + Headscale или Yggdrasil.
- Если у тебя один или несколько always‑on хостов: можно поднять ncps/nginx или self‑hosted signed cache.

Сравнение (кратко)

- Простота развертывания: Avahi > Tor > Yggdrasil > WireGuard+Headscale > Dynamic DNS
- Приватность: Tor > Yggdrasil/Headscale (private) > Dynamic DNS
- Latency / throughput: WireGuard/Headscale > Yggdrasil > Tor
- Требования к infra: Dynamic DNS (router) / Headscale (control plane) / Tor (none)
