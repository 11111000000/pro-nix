% pro-packages: specification

This document specifies the `pro-packages` runtime behaviour for the
pro-nix Emacs layer. It describes the early facts source, the prompt-and-install
policy, the decisions file, and the minimal integration points for modules.

## Files

- `~/.config/emacs/provided-packages.el` (generated by Nix)
  - Contains a single sexp: `(setq pro-packages-provided-by-nix '(pkg1 pkg2 ...))`
- `~/.config/emacs/decisions.el` (user decisions)
  - Contains: `(setq pro-packages-decisions '((pkg . always) (pkg2 . never)))`

## Behaviour

1. Site init loads `provided-packages.el` early.
2. During module configuration, modules call `pro-packages--maybe-install`
   before calling `use-package` for packages that may not exist.
3. If package is missing and policy allows MELPA, the user is prompted:
   - `i` = install once
   - `a` = always install (records "always")
   - `s` = skip forever (records "never")
   - `c` = cancel (skip this time)
4. Decisions are stored in `~/.config/emacs/decisions.el`.
5. In noninteractive mode the prompt is skipped; installs are allowed only if
   `PRO_PACKAGES_AUTO_INSTALL=1` is present in env.

## API (functions to be implemented)

- `pro--package-provided-p (pkg)` -> t/nil
- `pro-packages--load-decisions` and `pro-packages--save-decisions`
- `pro-packages--ask-user (pkg)` -> symbol of choice
- `pro-packages--do-install (pkg)` -> t/nil
- `pro-packages--maybe-install (pkg allow-melpa)` -> t/nil indicates package is available now

## Integration pattern for modules

Use the following pattern in modules that declare packages:

```el
(when (pro-packages--maybe-install 'gptel t)
  (use-package gptel
    :ensure t
    :config ...))
```

This pattern ensures the user is prompted before an automatic install.

## Logging

Install/skip decisions and summary should be logged to
`~/.config/emacs/pro-packages.log` with a compact per-session summary.
