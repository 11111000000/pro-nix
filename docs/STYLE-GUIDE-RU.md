Стиль — Руководство по текстам и комментариям (RU)
===============================================

Кратко
------
Язык: русский. Тон: точный, сжатый, учебный. Цель текста — описать контракт, ограничения и способ проверки.

Правила для README / HOLO / SURFACE
-----------------------------------
- README: компактный учебник — цель, quickstart (3–6 команд), основные инварианты, Contribution (Change Gate).
- HOLO: формальные инварианты и решения. Каждое решение: Pressure / Exit / Proof / Migration (если требуется).
- SURFACE: записи с полями: имя, стабильность, спецификация (1–2 предложения), Proof (команда), Owner (модуль/скрипт).

Комментирование кода
--------------------
- Язык: русский. Короткий контракт в начале файла/функции (1–2 строки). Если поведение нетривиально — краткая причина/рационал.
- Для публичных функций: docstring обязательный (Elisp, Python). Пример: "Выполняет X. Вход: Y. Побочный эффект: Z. Тест: tests/..."
- Не менять runtime-строки, которые являются частью CLI/интерфейсов; переводить только комментарии и документацию.

Тон и стиль
-----------
- Избегать субъективных слов без операции ("красиво", "удобно").
- Термины определяются при первом упоминании.

Процедуры изменений
-------------------
- Перед изменением публичного поведения: обновить SURFACE.md и добавить Proof.
- Для изменений документации сделать отдельный PR, если меняются форматы, тесты или Proofs.

Входные данные: описание типов/ограничений.
Возвращает: что возвращается.
Побочные эффекты: перечислить.
Тесты: tests/... (если есть)
"""
