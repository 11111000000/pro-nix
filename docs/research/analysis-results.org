* Анализ современных техник организации Emacs Lisp
  :PROPERTIES:
  :ID: modern-elisp-techniques-analysis
  :END:

** Введение

Этот документ содержит результаты анализа конфига pro-nix с точки зрения
современных техник организации Emacs Lisp кода для Emacs 30.2+.

Анализ показал, что конфиг уже в значительной степени следует современным
практикам, но есть возможности для улучшения.

** Результаты анализа

** 1. Лексическая связанность: ✓ Отлично

Все 23 модуля используют (require 'lexical-binding: t) в заголовках.

Пример из core.el:
```elisp
;;; core.el --- ядро Emacs -*- lexical-binding: t; -*-
```

** 2. Структура модулей: ✓ Хорошо

- Каждый модуль в отдельном файле
- Используется (provide 'module-name)
- Есть (require 'subr-x) для расширенных функций

Минусы:
- Нет (require 'cl-lib) для макросов Common Lisp
- Нет (require 'nadvice) дляAdvice-функционала

** 3. Пакетная загрузка: ✓ Хорошо

Используется условная загрузка через (require feature nil t) - это лучшая
практика для опциональных зависимостей.

Пример из ui.el:
```elisp
(when (require 'consult nil t)
  (consult-ripgrep ...))
```

** 4. Отложенная загрузка: △ Средне

Найдено 5 использований with-eval-after-load:
- pro-project.el
- keys.el  
- exwm.el
- org.el

Замечания:
- Можно использовать (:after feature) в use-package если переходить на него
- Нет использования delay-load для тяжелых пакетов

** 5. Настройка переменных: △ Средне

Используется defcustom для пользовательских переменных:
```elisp
(defcustom pro-ui-code-font-family "Fira Code"
  "Шрифт для кода."
  :type 'string
  :group 'pro-ui)
```

Минусы:
- Нет (:group 'module-name) для defcustom
- Некоторые переменные используют просто setq вместо defcustom

** 6. Функции и макросы: △ Средне

Найдено 64 defun, но:
- Нет cl-defun для keyword arguments
- Нет cl-labels для локальных рекурсивных функций
- Нет cl-macrolet для локальных макросов

** 7. Современные пакеты: ✓ Хорошо

Конфиг уже использует современные пакеты:
- vertico - для completion UI
- orderless - для matching
- marginalia - для annotations
- consult - для interactive commands
- corfu - для in-buffer completion
- eldoc-box - для documentation

** Рекомендации по улучшению

** Приоритет 1: Минимальные изменения

1. Добавить (require 'cl-lib) в core.el
   - Для cl-labels, cl-macrolet, cl-defun

2. Добавить cl-lib usage в keys.el
   - Заменить pcase на cl-case там, где применимо
   - Использовать cl-loop вместо while для сложной итерации

** Приоритет 2: Производительность

3. Добавить компиляцию:
```elisp
(setq comp-deferred-compilation t)
(setq comp-async-report-warnings-errors nil)
```

4. Использовать delay-load для больших пакетов:
```elisp
(delay-load 'some-big-package "2 seconds")
```

** Приоритет 3: Современные практики

5. Использовать cl-defun для функций с keyword args:
```elisp
(cl-defun my-func (&key (a 1) (b 2))
  ...)
```

6. Использовать cl-labels для локальных рекурсивных функций
7. Добавить (:after ...) для with-eval-after-load

** Сравнение с современными стандартами

| Техника                 | Текущее состояние | Стандарт 2026 |
|------------------------|-------------------|--------------|
| lexical-binding        | ✓ 100%            | ✓ 100%       |
| cl-lib                 | ✗ 0%             | ✓ 100%       |
| with-eval-after-load   | △ 5 uses          | △ 5-10 uses   |
| defcustom with :group  | △ 30%            | ✓ 100%       |
| cl-defun               | ✗ 0%             | ✓ 50%+       |
| vertico + consult      | ✓ Есть           | ✓ Есть       |
| delay-load             | ✗ 0%             | ✓ Рекомендуется |

** Заключение

Конфиг pro-nix уже следует многим современным практикам. Основные
улучшения:

1. Добавить require 'cl-lib
2. Использовать cl-defun/cl-labels для сложных случаев
3. добавить компиляцию через comp-deferred-compilation
4. Использовать delay-load для тяжелых пакетов

Это позволит сделать конфиг быстрее и соответствующим современным
стандартам Emacs 30.2+.
