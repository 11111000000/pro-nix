## HDS Workflow

1. Write the rule in text first.
2. Keep each Emacs module focused on one concern.
3. Treat Org tables as the source of truth for declarative surfaces.
4. Recompile text with `org-babel-execute:org` when the surface changes.
5. Apply Emacs-layer changes with `just install-emacs`.

## Emacs Lisp Style

1. Keep functions small, explicit, and easy to read.
2. Prefer one module per concern.
3. Use Org for declarative surfaces, not scattered ad hoc strings.
4. Make load order explicit when a module depends on another.
5. Treat text as the testable contract when the config is generated by an LLM.

## Keybinding Workflow

1. Edit `emacs-keys.org` for new keybindings.
2. Use `org-babel-execute:org` to generate Emacs Lisp.
3. For local overrides, edit `~/.emacs.d/keys.org`.
4. Run `just install-emacs` to apply changes.

> Keybindings compile to `~/.emacs.d/keys.el` and load on startup.
