### HDS Rules

- Read `AGENTS.md` first, then `SURFACE.md`, then `README.md` before editing.
- Text is test: if a rule matters, write it and keep it checkable.
- One file, one concern: prefer small modules with explicit contracts.
- Prefer explicit ordering over hidden coupling.
- Do not invent new abstractions unless they reduce complexity.

### Emacs Lisp Rules

- Keep functions small and named by role.
- Prefer explicit contracts over clever control flow.
- Use `use-package` for external packages and plain `defun`/`setq` for local policy.
- Make load order explicit when a module depends on another.
- Treat text as the testable contract when config is generated by an LLM.

### Keybinding Policy

- All keybindings must be defined in `emacs-keys.org`.
- Compile with `org-babel-execute:org`.
- Overrides go in `~/.emacs.d/keys.org` with `:org` prefix.
- After changes, run `just install-emacs`.

> Keybindings are loaded from `~/.emacs.d/keys.el` on startup.
